## Steps 

Phase 1 - 页面骨架重构

将单栏页面改为两栏：左侧图谱，右侧侧边栏。

侧边栏拆成四个区块：分区控制、搜索、添加节点、添加关系与保存。

该阶段完成后，后续功能统一挂载到侧边栏事件流。

# Phase 2 - 图数据增强与分区显示

在图构建阶段计算弱连通分量，为节点和边添加 component_id。

前端按 component_id 做并排网格偏移布局，让不相关图谱自动分区。

提供分区显隐控制（全选/单选/反选），支持按分量过滤显示。

# Phase 3 - 节点搜索过滤

增加搜索输入与实时匹配。

搜索结果采用过滤显示：仅展示匹配节点及一跳邻接关系。

提供“清空恢复全图”，并与分区过滤联动（两类过滤取交集）。

# Phase 4 - 新增节点与关系

添加节点表单：名称与可选描述，写入 DataSet 即时渲染。

添加关系表单：起点、终点、关系名、方向类型。

关系支持有向/无向；无向关系在渲染和连通性计算中按无向处理。

新边连接不同分区时，增量合并分量并重排布局。

# Phase 5 - 保存能力

先实现前端导出（JSON/CSV）保存当前编辑结果。

预留后续写回 CSV 的接口位（下一轮可接 Python 导入或本地服务）。

# Phase 6 - 健壮性与验收

表单校验：空值、重名、重复边、自环策略。

节点数、边数、分量数实时更新。

完成针对分区、搜索、创建、导出的回归验收。

## Relevant files

MindMap.py - 主改造文件：模板布局、前端交互脚本、连通分量计算与数据注入。
knowledge_graph_interactive.html - 生成结果文件，用于交互验收。
Stage_Tech_MindSet.csv - 原始三元组输入，后续落盘时需要兼容该结构。
Verification

使用两组互不相连三元组，确认自动并排分区。
搜索关键词，确认过滤范围正确；清空后恢复全图。
新增节点和有向/无向关系，确认即时渲染与方向语义正确。
连接两个独立分区，确认自动合并与重排。
导出 JSON/CSV，确认内容可复现当前图状态。
